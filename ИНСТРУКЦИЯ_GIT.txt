  Git установлен, но не найден в PATH
═══════════════════════════════════════════════════════════════

Git установлен, но PowerShell его не видит. Это нормально после
установки - нужно перезапустить терминал или добавить в PATH.

═══════════════════════════════════════════════════════════════
  РЕШЕНИЕ 1: ПЕРЕЗАПУСТИТЬ POWERSHELL
═══════════════════════════════════════════════════════════════

1. Закройте текущий PowerShell
2. Откройте новый PowerShell (от имени администратора)
3. Перейдите в папку проекта:
   cd C:\Users\super\telegram-shipping-bot
4. Запустите скрипт:
   .\git_upload_fix_path.ps1

═══════════════════════════════════════════════════════════════
  РЕШЕНИЕ 2: ИСПОЛЬЗОВАТЬ СКРИПТ С ПОИСКОМ GIT
═══════════════════════════════════════════════════════════════

Запустите скрипт, который сам найдет Git:

1. Откройте PowerShell в папке проекта
2. Выполните:
   powershell -ExecutionPolicy Bypass -File git_upload_fix_path.ps1

Скрипт автоматически найдет Git в стандартных местах.

═══════════════════════════════════════════════════════════════
  РЕШЕНИЕ 3: ЗАГРУЗИТЬ ЧЕРЕЗ ZIP-АРХИВ
═══════════════════════════════════════════════════════════════

Если Git не работает, используйте готовый архив:

1. Откройте: https://github.com/bryanwax12/newbotcursor
2. Add file → Upload files
3. Перетащите: C:\Users\super\telegram-shipping-bot.zip
4. Commit changes

═══════════════════════════════════════════════════════════════
  РЕШЕНИЕ 4: ИСПОЛЬЗОВАТЬ GIT BASH
═══════════════════════════════════════════════════════════════

Если установлен Git, должен быть Git Bash:

1. Откройте Git Bash (из меню Пуск)
2. Выполните команды:

   cd /c/Users/super/telegram-shipping-bot
   git init
   git add .
   git commit -m "Initial commit"
   git remote add origin https://github.com/bryanwax12/newbotcursor.git
   git branch -M main
   git push -u origin main
